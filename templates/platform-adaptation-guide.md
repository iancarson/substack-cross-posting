# Platform Adaptation Guide

How to reformat Substack content for LinkedIn, Medium, and X without losing meaning or wasting time.

---

## The Core Principle

Each platform has a different reader expectation and algorithmic environment. The same content needs different framing, not different ideas.

**You're not rewriting — you're reframing.**

---

## Adapting Substack Notes for LinkedIn

### What changes

| Element | Substack Note | LinkedIn Version |
|---|---|---|
| Opening line | Conversational, assumes Substack context | Self-contained hook, professional framing |
| Tone | Casual, direct | Professional but human |
| References | Can mention Substack-specific things (restacks, Notes, etc.) | Remove all Substack jargon |
| Length | 100–300 characters | Up to 1,200–1,500 characters |
| CTA | Optional | Always: "Subscribe at [URL]" |
| Links | Inline | In first comment (if reach matters) |

### Step-by-step adaptation

**1. Rewrite the opening line**

Substack: *"The thing nobody talks about in newsletter growth..."*
LinkedIn: *"Most newsletter writers plateau at 500 subscribers and assume it's a content quality problem. It's not."*

LinkedIn's algorithm rewards posts that grab in the first line. Make it specific and standalone.

**2. Add one sentence of context**

LinkedIn readers may not know your newsletter or niche. Add 1 sentence that grounds the topic.

**3. Keep the body mostly the same**

The insight doesn't need to change. The setup does.

**4. Add a subscribe CTA**

Every LinkedIn cross-post should end with:
*"I cover this weekly in [Newsletter Name] — subscribe free at [URL]"*

**5. Move links to the first comment**

LinkedIn's algorithm suppresses posts with external links in the body. Post the body first (link-free), then add your Substack link in the first comment immediately after publishing.

## Adapting Substack Notes for X (Twitter)

### Short Notes (under 280 chars)

Post as-is if it works as a standalone tweet. If it needs Substack context, trim and rewrite.

### Medium Notes (280–600 chars)

Convert to a 2–3 tweet thread:
- Tweet 1: The hook (strongest line)
- Tweet 2: The explanation or evidence
- Tweet 3: The implication + CTA with link

### Longer Notes / Articles

Convert to a 5–8 tweet thread:
- Tweet 1: The contrarian hook or main claim
- Tweets 2–6: One point each, numbered "2/ ... 3/ ..."
- Tweet 7–8: Synthesis + link to full piece

**Example thread structure:**
```
[1/6] [Hook: the most surprising/contrarian version of your main point]

[2/6] [Context: why this matters now]

[3/6] [First supporting point]

[4/6] [Second supporting point]

[5/6] [Third supporting point / turning point]

[6/6] [Conclusion + CTA]

Full breakdown in this week's newsletter: [link]
```

## Adapting Substack Articles for Medium

### What changes

| Element | Substack Article | Medium Version |
|---|---|---|
| Canonical URL | Original | Set to Substack URL |
| Subscribe CTA | Email capture (native) | Add manual CTA at end |
| Images | Substack CDN | Re-upload to Medium |
| Tags | N/A | Add 3–5 Medium tags |
| Internal links | Substack-native | Keep (they work cross-site) |

### Step-by-step adaptation

**1. Copy the full article text**

Copy everything — headings, body, pull quotes.

**2. Re-upload images**

Medium won't embed Substack CDN images reliably. Download and re-upload.

**3. Set canonical URL**

In Medium's Advanced Settings: paste your Substack article URL as the canonical URL.
This is critical for SEO — it tells Google which version is the original.

**4. Add subscribe CTA at the end**

```
---
*If you found this useful, I write about [topic] every week.
Subscribe free at [Substack URL] →*
```

**5. Add Medium tags**

Choose 3–5 tags that match your article's topic. This determines which Medium readers see your article in their feed.

**6. Check formatting**

Medium renders Markdown differently than Substack. Review headings, blockquotes, and code blocks before publishing.
